# Product Requirements Document (PRD)

## CalcWise — Free Online Finance Calculators & Tools

---

## 1. Executive Summary

**CalcWise** is a free, ad-supported website offering 100+ personal finance calculators, budgeting tools, and financial education resources. It targets the massive evergreen demand for financial calculations (mortgage payments, loan interest, tax estimates, retirement planning, etc.) and monetizes through Google AdSense and display ads.

**Why this works:**

- Finance keywords have the **highest CPC in Google Ads** ($5–$50+ per click)
- Calculator queries have **enormous, consistent search volume** (e.g., "mortgage calculator" = 5M+ monthly searches in the US alone)
- Tools are **evergreen** — people will always need to calculate loans, taxes, budgets
- Each calculator is a **standalone SEO landing page**, creating hundreds of high-ranking entry points
- Users have **real intent** — they're actively making financial decisions
- Competitor benchmark: calculator.net receives **80M+ monthly visits** and earns an estimated **$500K–$2M/month** in ad revenue

---

## 2. Problem Statement

People making financial decisions (buying a home, taking a loan, planning retirement, filing taxes) need quick, accurate, free calculations. Banks and financial institutions bury these tools behind sign-up walls or bias results toward their products. Generic search results provide formulas but not interactive tools.

**CalcWise solves this** by providing instant, unbiased, beautifully designed financial calculators with zero sign-up required.

## 8. Technical Architecture

### 8.1 Tech Stack

| Layer            | Technology                                 | Rationale                                                    |
| ---------------- | ------------------------------------------ | ------------------------------------------------------------ |
| **Framework**    | Next.js 14+ (App Router)                   | SSR/SSG for SEO, React for interactivity, API routes         |
| **Language**     | TypeScript                                 | Type safety for financial calculations                       |
| **Styling**      | Tailwind CSS                               | Rapid UI development, small bundle                           |
| **Charts**       | Chart.js or Recharts                       | Interactive, responsive financial charts                     |
| **Hosting**      | Vercel                                     | Zero-config Next.js deployment, edge CDN, free tier to start |
| **Analytics**    | Google Analytics 4 + Google Search Console | Traffic tracking, SEO monitoring                             |
| **Ads**          | Google AdSense + Google Ad Manager         | Primary revenue                                              |
| **CMS (Blog)**   | MDX files or Sanity.io                     | SEO-friendly blog content                                    |
| **Currency API** | ExchangeRate-API (free tier)               | Real-time currency conversion                                |
| **Tax Data**     | Static JSON files updated annually         | Federal/state tax brackets                                   |

### 8.2 Performance Requirements

- **Lighthouse Score:** 90+ on all metrics (Performance, Accessibility, Best Practices, SEO)
- **First Contentful Paint:** < 1.5s
- **Largest Contentful Paint:** < 2.5s
- **Cumulative Layout Shift:** < 0.1 (critical for ad-heavy pages)
- **Time to Interactive:** < 3s
- **Core Web Vitals:** All "Good" (required for Google ranking)

### 8.3 SEO Technical Requirements

- Server-side rendered or statically generated pages
- Dynamic sitemap.xml generation
- robots.txt properly configured
- Structured data (JSON-LD) on every page
- Canonical URLs on all pages
- Responsive design (mobile-first)
- HTTPS everywhere
- Fast TTFB via edge caching

## 9. Design Requirements

### 9.1 Design Principles

- **Clean & trustworthy** — financial tools need to feel reliable (think: government website meets modern SaaS)
- **Utility-first** — calculator front and center, no clutter
- **Mobile-optimized** — 60%+ traffic will be mobile
- **Accessible** — WCAG 2.1 AA compliant
- **Fast** — perceived speed matters for bounce rate

### 9.2 Color Palette

- Primary: Deep blue (#1a365d) — trust, finance, professionalism
- Secondary: Teal/green (#38a169) — money, growth, positive results
- Accent: Amber (#d69e2e) — call-to-action highlights
- Background: Light gray (#f7fafc) — clean, easy on the eyes
- Text: Dark gray (#2d3748) — high readability

### 9.3 Typography

- Headings: Inter or Plus Jakarta Sans (modern, clean)
- Body: Inter or system font stack (performance)
- Numbers/Results: Monospace or tabular figures for alignment

### 9.4 Key UI Components

- **Calculator Card** — self-contained widget with inputs, calculate button, results
- **Results Panel** — highlighted section with large numbers, breakdown table
- **Chart Section** — interactive visualization of results
- **Category Navigation** — icon-based category cards on homepage
- **Search Bar** — prominent search on homepage ("What do you want to calculate?")
- **Breadcrumbs** — every page for navigation and SEO
- **Related Calculators Carousel** — keep users on-site
- **"Was this helpful?" widget** — user feedback for iteration
